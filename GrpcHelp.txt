Структура проекта gRPC:

Папка Protos содержит файлы с определением сервисов и сообщений, используемых для взаимодействия по сети

По умолчанию в этой папке определен один файл greet.proto.

Папка Services содержит файлы с реализацией сервисов

По умолчанию в этой папке определен один файл GreeterService.cs.

Файл appsettings.json - стандартный файл конфигурации проекта в ASP.NET Core.

Файл Program.cs содержит стандартный класс Program, с которого начинается выполнение приложения ASP.NET Core.

Файл Startup.cs содержит стандартный класс Startup приложения ASP.NET Core.

Определение сервиса
Таким образом, мы видим, что по сравнению с обычными проектами ASP.NET Core здесь 
два новый элемента - каталоги Protos и Services. gRPC использует подход "contract-first", 
то есть вначале определяется контракт - общее определение сервиса, которое определяет механизм взаимодействия.
